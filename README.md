# CareerFlow

一个面向求职场景的 AI 求职辅助工具。

CareerFlow 主要聚焦求职过程中几个高频但低效的问题：岗位要求难拆解、简历与岗位匹配度不清晰、面试准备缺乏针对性、投递过程分散难管理。围绕这些问题，我设计并搭建了一个覆盖岗位解析、简历优化、面试训练与投递管理的求职辅助 Demo，希望通过 AI 能力帮助用户更高效地完成从“看岗位”到“准备投递”的完整流程。

## 项目背景

在实际求职过程中，很多信息并不是没有，而是分散、冗长且难以快速转化为行动：
- 岗位 JD 信息量大，但重点职责、核心能力要求不够清晰
- 简历修改往往停留在泛化建议，缺少针对具体岗位的分析
- 面试准备容易流于通用问题，和目标岗位、个人经历结合不够紧密
- 投递过程中的岗位信息、修改版本和准备进度缺少统一管理

CareerFlow 试图解决的不是单一环节问题，而是把岗位分析、简历修改、面试准备和投递管理串成一个更完整的求职辅助流程。

## 核心功能

### 1. 岗位解析
围绕岗位 JD 进行结构化拆解，提取岗位职责、任职要求和可能重点考察的能力项，帮助用户更快理解岗位信息，而不是停留在逐句阅读。

### 2. 简历优化
结合岗位要求和用户简历内容，生成更有针对性的修改建议，帮助用户识别已有匹配项、缺失项以及可强化表达的部分。

### 3. 面试训练
基于目标岗位和简历信息生成更有针对性的面试问题，帮助用户围绕真实投递场景做准备，而不是仅依赖通用题库。

### 4. 投递管理
将岗位信息、简历修改、准备状态等内容纳入同一流程，减少信息割裂，提高求职过程的可追踪性。

## AI 设计思路

CareerFlow 并不是简单接入模型输出结果，而是围绕具体场景拆分任务，分别设计对应的 Prompt，用于支持不同模块的能力输出，例如：
- JD 分析
- 简历优化建议生成
- 面试问题生成

在实际迭代过程中，我重点关注了生成结果的相关性、结构化程度和可用性，并针对建议生成过程中出现的内容编造问题，对 Prompt 方案进行了持续调整与优化，使结果尽量建立在用户输入内容和岗位信息之上，而不是生成脱离语境的泛化建议。

## 项目亮点

### 场景完整
不是只做单点功能，而是将岗位解析、简历优化、面试训练和投递管理串联起来，形成相对完整的求职辅助链路。

### AI 能力和产品流程结合
项目重点不只是“接了模型”，而是把模型能力放进具体的用户任务里，围绕实际使用场景设计输入、输出和展示方式。

### 有明确的迭代意识
在使用过程中，不仅关注“能不能生成”，也关注“生成得是否有用”。对于建议生成中出现的不准确和编造问题，我通过调整 Prompt 方案持续优化结果质量。

### 可交互、可展示
项目已经完成 Demo 搭建并上线，具备对外展示和进一步迭代的基础。
